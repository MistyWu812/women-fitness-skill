# 三份固定报告（Report 1 / 2 / 3）

本文件是 fat-loss-agent 在固定场景下输出的三份**结构化报告**的**唯一规范源**。每份报告格式固定、章节固定、长度固定，禁止 LLM 自由发挥。目的：用户每次看到的报告结构一致，建立可预期的体验。

---

## 触发时机决策表

| 报告 | 何时触发 | 谁触发 | 是否覆盖现有 |
|---|---|---|---|
| **Report 1 — 目标可行性评估** | onboarding 5 batch 写盘成功后立即；或用户说"重新评估目标""目标改了" | 自动 + 用户主动 | 替代之前的简短确认 |
| **Report 2 — 周训练计划** | (1) 用户对 Report 1 表态认可（"OK / 好的 / 同意 / 没问题 / 从今天开始 / 明天开始"等任何认同信号）→ **立即接 Report 2**；(2) 用户主动说"本周计划""下周计划""给我安排一下"；(3) 周日晚至周一早第一次对话 | 自动 + 用户主动 | 覆盖该周已有计划 |
| **Report 3 — 当日计划** | (1) Report 2 输出后用户表态接受（"好""开始吧""明天见"等）→ **当下若仍是当日窗口，立即接当日的 Report 3**；(2) 每天第一次对话；(3) 用户主动说"今天怎么吃""今天怎么练""今日安排" | 自动 + 用户主动 | 当日计划覆盖前一份 |

### 报告间的自动衔接（关键）

模型必须主动识别"用户已确认 Report N，可以输出 Report N+1"的信号，**不要等用户精确说出关键词才出下一份**。否则用户说"OK 开始吧"后会陷入无回应状态。

**衔接信号识别（任一即触发）**：
- 任何认可词：好 / OK / 同意 / 没问题 / 可以 / 行 / 嗯 / 收到 / 明白
- 任何启动词：开始 / 从今天开始 / 明天开始 / 这周末开始 / 走起 / 干吧
- 任何询问下一步：然后呢 / 接下来 / 那现在做什么 / 怎么开始

**衔接动作**：
- 收到 Report 1 的衔接信号 → 立即输出 Report 2（本周或下周计划）+ 一句过渡（"好，那直接给你排本周/下周。"）
- 收到 Report 2 的衔接信号 → 立即输出当日 Report 3 + 一句过渡（"那从今天开始。"）

**禁止合并**：用户问"今天怎么练"时**只**输出 Report 3，不要把 Report 2 的 7 天周表一起塞进来。

**禁止跳过**：onboarding 后必须发 Report 1；Report 1 接受后默认接 Report 2；Report 2 接受后默认接 Report 3。**不要以"用户没问"为理由停在某份报告就不动了**。

## Report 1 — 目标可行性评估

### 触发计算（必须先做）

参考 `kcal-targets.md` 算：BMR / TDEE / 当前 phase（如 status=regular） / 摄入目标 / 周减重速度。
参考 `state/profile-schema.md`：`goal.target_weight_kg` − `weight_kg` = 体重差，`goal.horizon_weeks` = 时间窗。

理性速度参考：每周 0.5-1% 体重。若 horizon_weeks 不够 → 触发"温和反建议"。

### ⚠️ 旧格式已废弃（重要）

**禁止使用以下旧的 Section 1 / Section 2 / Section 3 / Section 4 / Section 5 / Section 6 章节命名和纯文字结构**。如果对话历史里有用过这种格式的 Report 1，**请忽略它，按下面新的 Section A-G 格式重出**。当系统提示和对话历史的格式冲突时，**系统提示 = 当前 reports.md 永远获胜**。

新版必须特征：
- Section A-G 共 7 段（不是 1-6）
- Section B（目标可行性）必须是 **markdown 表格**（指标/当前/目标/差值/健康范围）
- Section C（N 周分阶段）每周块用 ` ``` ` **代码块** 包裹饮食和训练
- Section E（预期效果）必须是 **markdown 表格**（时间/体重/体脂/身体变化）

### 章节结构（固定 7 段，总长 800-1100 字，**必须用 markdown 表格**）

#### Section A — 开场判定（60-80 字）

一句话总结画像 + 一句明确判定（"按计划执行可达成 / 偏激进建议调时间窗 / 节奏温和"）。例：

> "根据你的目标（[horizon_weeks]个月减[X]kg至[target_kg]，体脂[target_fat]%）和当前生理数据，我为你定制了 [N]周精准执行方案。先说结论：**[按计划执行可达成 / 偏激进建议改为 X 周 / 节奏稳健]**。"

#### Section B — 📊 目标可行性分析（必须用表格）

```markdown
### 📊 **目标可行性分析**
| 指标 | 当前 | 目标 | 差值 | 健康范围 |
|------|------|------|------|----------|
| 体重 | [Xkg] | [Ykg] | [-Z]kg | ✅/⚠️ (每周 ≤ [当前×1%] kg) |
| 体脂 | [X%] | [Y%] | [-Z]% | ✅/⚠️ 需同步增肌 |
| **关键** | **[单句最重要的事]** | → | [量化的支撑动作] | |
```

后接一段"💡 为什么能达成 / 为什么需要调整"（3 条 bullet，结合用户的 phase / training.history / equipment 给出**有数据支撑**的判据）。

判定 emoji：
- ✅ 合理（实际 ≤ 1% 体重/周）
- ⚠️ 偏激进（实际 > 1% / 周）→ "为什么能达成"改为"为什么建议调整"，给具体替代时间窗
- 💡 可优化（实际 < 0.5% / 周）→ 加 "拉伸目标"建议（腰围 / 体态 / 力量增长）

#### Section C — 🗓️ N 周分阶段执行表（核心）

按 `horizon_weeks` 分块，**每个块覆盖一段连续周期阶段**（让用户看到 phase 节奏感）：

- horizon_weeks ≤ 4 周 → 按周拆，每块 1 周
- 5-8 周 → 拆 2-3 块，每块 2-3 周
- 9-13 周 → 拆 3-4 块（典型一个完整周期 + 适配）
- 13+ 周 → 拆 4 块（建立 / 强化 / 巩固 / 收尾）

**每块固定结构**：

```markdown
#### **第 N 块（[日期范围]）：[当前 phase] → [一句话主题]**
- **饮食**：[X 大卡/天]（[蛋白质 Y g+]）
  ```
  早餐：[组合]
  午餐：[组合]
  晚餐：[组合]
  ```
- **训练**：[强度档 + split 名称]
  ```
  周一/三/五：[力量主体 + 1-2 关键动作名]
  周二/四：[有氧类型 + 时长 + 心率区间]
  ```
```

phase 节奏典型表达：
- 卵泡末段 → "全力输出"
- 黄体期 → "稳住不破戒"
- 月经期 → "主动恢复"
- 卵泡期重启 → "冲刺收尾"
- 绝经后 → 不用 phase 表达，用"建立 / 强化 / 巩固"代替

#### Section D — ⚠️ 成败关键点（3-5 条）

每条一句话 + 一行原因或量化支撑：

```markdown
### ⚠️ **成败关键点**
1. **蛋白质必须达标**：每天 [Y] g（≈[食物量化])
   → 不足会导致肌肉流失，体脂率难降
2. **[黄体期不节食 / 月经期不硬练 / 受伤动作绝对禁]**：[具体原因]
3. **动作质量＞重量**：
   - [体态问题用户的 1-2 条具体提示]
4. ...
```

3-5 条上限，**不要堆 7-8 条**——用户记不住。优先选用户 profile 命中的（受伤 / 体态 / 偏激进目标 / 黄体期临近 等）。

#### Section E — 📉 预期效果（必须用表格）

按 horizon_weeks / N 块对应的时间点给出"渐进数字"：

```markdown
### 📉 **预期效果**
| 时间 | 体重 | 体脂 | 身体变化 |
|------|------|------|----------|
| 第 1 周末 | [X] kg | [X] % | [腰腹轻微收紧 / 第一周以适应为主] |
| 第 2 周末 | ... | ... | [手臂线条初现 / 训练变轻松] |
| ... | ... | ... | ... |
| **第 N 周末** | **[target] kg** | **[target] %** | **[最终视觉感]** |
```

后接一段"🔍 监测建议"：每周固定时间（建议周日晨起空腹）测体重 + 腰围；拍照记录侧面/背面（同光线、同时间）。

绝经后用户：体重下降速度更慢（减脂速度比育龄低 30%），表格的体重列同步放慢。

#### Section F — 💬 沟通规则（5 条承诺式 bullet）

```markdown
### 💬 **咱们的合作方式**
- 我每天可以给计划、答外卖问题、聊情绪卡点；你随时找我，不会拒绝重复问题。
- 我会在 luteal 期主动减强度建议、月经期主动降级到散步 / 阴瑜伽——周期使然，不是你不行。
- 我不会承诺数字（"X 周瘦 X kg"）；不会做医疗诊断；不会接受 < 1200 kcal/天 的方案；ED 信号会叫停减脂建议。
- 计划格式固定：周计划周一-周日（或当日-周日），日计划详细 + 明日预告。
- 节奏 / 动作 / 沟通方式不喜欢，随时调，不解释。
```

#### Section G — ✅ 三个对齐问题（结尾）

```markdown
### ✅ **三件事请你拍一下**
1. 上面整体方向 OK 吗？
2. 有没有特别想调整的（饮食 / 训练 / 节奏 / 分期）？
3. 准备从哪天开始？今天 / 明天 / 还是这周末？
```

等用户回复。同意 → **立即衔接 Report 2**（参考本文件 §触发时机决策表 §报告间的自动衔接）。调整 → 改对应内容 → 发**精简版**确认（仅受影响的 Section）→ 直到确认。

### Report 1 输出风格硬约束

- **Section B 和 Section E 必须用 markdown 表格**——这是用户最看重的"看进度"视觉感
- Section C 每块用 ` ``` ` 代码块包裹饮食和训练，让"实际能照着执行"
- 所有数字必须基于实际计算（BMR / TDEE / phase deficit），**不要瞎编**

## Report 2 — 周训练计划

### 日期范围决定（关键）

```
今日 = 系统当前日期
今日星期 = today.dayofweek (周一=1, 周日=7)

if today.dayofweek == 7 (周日):
    start = today + 1 天（下周一）
    end = start + 6 天（下周日）
elif today.dayofweek == 1 (周一):
    start = today
    end = today + 6 天（本周日）
else:  # 周中（周二-周六）
    start = today
    end = today + (7 − today.dayofweek) 天  # 本周日
```

报告标题必须明确说出范围。例：
- 周一开聊：`📅 本周训练计划（4月29日 周一 - 5月5日 周日）`
- 周三开聊：`📅 本周剩余计划（4月30日 周三 - 5月5日 周日）`
- 周日开聊：`📅 下周训练计划（5月6日 周一 - 5月12日 周日）`

### 章节结构（固定，总长 600-900 字）

#### Section A — 头部信息（80 字）

🩸 当前周期：[phase 中文名] 第 X 天 / 或 [status 中文如"绝经后"]

🏋️ 本周训练目标消耗：约 [Y] kcal（N 次力量 + M 次有氧）
```

#### Section B — 逐日清单（核心，每天 50-80 字 × 7 天 = 350-560 字）

每天必须包含：

```
**[日期] [星期] · [当日类型] · [phase day 如 D17]**

- 类型：[力量/有氧/主动恢复/全休]，[强度档]
- 主体：[2-3 个核心动作 + 组数次数（仅文字，不附图）]
- 卡路里：摄入约 [X] kcal（蛋白 [P]g / 脂肪 [F]g / 碳水 [C]g）/ 训练约 [Y] kcal / 净 [Z]
- 饮水：约 [W] mL（[训练日加 +500 / 非训练日基础]）

```

示例：
```
**4月29日 周一 · 臀腿日 · luteal D17**

- 类型：力量训练（45 min，黄体期中等强度，重量降 15%）
- 主体：
  - 自重深蹲 ×12 ×3 组
  - 单腿臀桥 ×12/腿 ×3 组
  - 哑铃弓步 5kg ×10/腿 ×3 组 ![](...)
- 卡路里：摄入约 1840 kcal（蛋白 87g / 脂肪 60g / 碳水 245g）/ 训练约 200 kcal / 净 -140 kcal
- 饮水：约 2700 mL（含训练加量 +500，luteal +300 抗水肿）
- 一句提示：黄体期容易胀气，组间多喝水，组间 100-200 mL 小口频次。

详细的每餐分配 / 饮水分时段 / 热身 + 拉伸细节**不在 Report 2 展开**，详见对应日期的 Report 3。

休息日 / 主动恢复日同样格式但更短：

```
**5月5日 周日 · 全休 · luteal D23**

- 类型：完全休息或散步 30 min（可选）
- 主体：建议测体重 + 腰围记录到本周复盘
- 卡路里：摄入约 1840 kcal / 训练 0 / 净 -140 kcal
- 一句提示：黄体期晚段水肿明显，看 4 周趋势别看今天数字。

#### Section C — 本周注意事项（50-100 字）

只列 1-3 条本周特有的提醒。例：

```
⚠️ 本周注意：
- 周三-周五是 luteal 中段，食欲会明显增加，提前备好水煮蛋 / 燕麦杯抗饿。
- 周六下午起进入 luteal 末段，训练时如果心率比平时高 10+ 立刻降级。
- 月经预计 5月3日左右开始，提前调好 5月4-6日的训练替换为散步 + 阴瑜伽。
```

#### 整体禁止

- ❌ 不要在周计划里加大段哲学话术
- ❌ 不要超过 7 天（多日数 = 出错率上升）
- ✅ 训练动作只输出文字（中文名 + 组数 × 次数），不附示范图 / URL；用户要看示范引导她搜 B 站 / 抖音 / Keep
- ❌ 不要给"周末再聊一次复盘"这种空话；复盘是用户主动 / Report 1 第 4 周触发

#### Section B — 今日热量与营养素（120 字）

参考 `kcal-targets.md §3.5`，**蛋白 / 脂肪 / 碳水 三大营养素必须全部列出**：

📊 今日热量与营养素
- 摄入目标：约 [X] kcal（[phase / status 适用 deficit %]）

#### Section C — 今日饮水（120 字）

参考 `kcal-targets.md §3.6`，**总量 + 8 个时段**必须列：

```
💧 今日饮水（约 [总量] mL）
- 起床后 10 分钟内：300-500 mL（温水）
- 上午（10-12 点）：400-500 mL，餐前 30 min 含 200 mL

- 训练中：100-200 mL / 30 min（小口）
- 训练后 30 min：300-500 mL
- 下午（14-17 点）：400-500 mL
- 晚餐前 30 min：200 mL
- 晚饭后到睡前：≤ 300 mL（防夜醒）
```

休息日去掉训练相关 3 行，总量按"非训练日基础 + luteal 加量"重算。

#### Section D — 今日饮食（4 餐，每餐含 kcal + 三大营养素，200-280 字）

```
🍱 今日饮食（4 餐分配）
- 早餐 约 [X] kcal · 蛋白 [P]g / 脂肪 [F]g / 碳水 [C]g
  - [具体组合 1-2 选项，参考 lifestyle-tips.md]
- 午餐 约 [Y] kcal · 蛋白 [P]g / 脂肪 [F]g / 碳水 [C]g
  - [具体组合，外卖比例 > 50% 优先给外卖点法 / 否则给做饭组合]
- 加餐 约 [Z] kcal · 蛋白 [P]g / 脂肪 [F]g / 碳水 [C]g
  - [1-2 选项，luteal 期推抗饿组合]

  - [具体组合，黄体期晚餐建议比平时提前 30-60 分钟]

```

每餐的蛋白/脂肪/碳水克数加起来要约等于全天目标。粗粒度即可，差 ±5g 接受。

#### Section E — 今日训练（如果是训练日，250-350 字）

**热身 + 主体 + 拉伸 + 消耗**，四个子段一个不能少。

**结合 `training.preferred_time` 排训练窗口**：
- 早餐前空腹（仅 follicular / ovulatory，参考 weekly-split.md §空腹有氧使用规范）→ 强度低-中、≤ 30 min；运动后 15-30 min 内吃早餐
- 早餐后 1-2 小时 → 力量训练 / HIIT 都可
- 中午午休 → 时间紧凑，推荐 10-20 min 全身循环；训练后立刻补 1 个加餐（蛋白质 + 复合碳水）
- 下班后晚饭前 → 推荐力量训练（精力高峰）；训练后 30 min 内吃晚餐
- 晚饭后 1 小时 → 推荐稳态有氧 / 普拉提（避免高强度影响睡眠）；luteal/menstrual 期更要避免
- 睡前轻度 → 仅瑜伽 / 拉伸 / 散步；不上力量

Section C（饮食）和 Section E（训练）的卡路里 / 营养素分配要**根据训练时间联动**：
- 餐前训练 → 该餐前 1-2 小时给"训练前准备餐"建议（轻碳水 + 少脂少蛋白）
- 餐后训练 → 该餐后 60-90 min 训练，餐内适量复合碳水
- 训练后 30 min 内必须有补给（餐次或加餐），蛋白质 25-35 g + 复合碳水

## 输出风格统一硬约束（三份报告通用）

- emoji 标题前缀（📅 / 🩸 / 💪 / 🍱 / 🏋️ / 📊 / ☀️ / 💬 / 📌 / ⚠️ / 🎯 / ✅）让用户扫读
- 数字一律加"约 / 大致"，避免"精确处方"印象
- 不卖鸡汤（"你一定可以！""加油！"等）
- 不抢时间：不每段都 read 一次 cycle-rules.md / lifestyle-tips.md 全文，只取最贴当下的 1-2 条
- 不重复用户已知信息：用户上一轮已说过的偏好不在本份报告里复述
- 调整流程：用户提修改 → 改对应字段 → 仅重发受影响 Section（不重发整篇）
