# 热量目标计算与展示

本文件给 LLM 在输出每日 / 每周计划时计算"饮食目标 kcal / 训练燃烧 kcal / 净赤字"，让用户每天能看到**进度数字**。

**所有数字都是估算**，输出给用户时必须用 "约 / 大致 / 估算"措辞，避免给人"精确到 1 kcal"的错觉。

## 1. BMR（基础代谢率）计算 —— Mifflin-St Jeor 女性公式

```
BMR (kcal/天) = 10 × weight_kg + 6.25 × height_cm − 5 × age − 161
```

例：35 岁女性，165 cm，58 kg →  
BMR = 10×58 + 6.25×165 − 5×35 − 161 = 580 + 1031 − 175 − 161 = **1275 kcal/天**

## 2. TDEE（每日总能耗）= BMR × 活动系数

LLM 按 `profile.activity_level` 取系数：

| activity_level | 系数 | 说明 |
|---|---|---|
| `sedentary` | 1.2 | 久坐 |
| `light` | 1.375 | 轻度走动 |
| `moderate` | 1.55 | 中等强度 / 每周 2-3 次训练 |
| `active` | 1.725 | 重体力 / 每周 4+ 次训练 |

例：上面的 BMR=1275，activity=moderate → TDEE ≈ 1275 × 1.55 ≈ **1976 kcal/天**

> **重要**：activity_level 已经包含了"一般训练量"。**计划日训练燃烧不再加到 TDEE 上重复计算**——只用作"今日实际燃烧多了多少"的参考。

## 3. 饮食目标 = TDEE − Deficit（按 phase 调整，仅 cycle.status=regular 走本表）

**`cycle.status ≠ regular` 的用户走以下表**：

| `cycle.status` | Deficit | 摄入目标 = TDEE × 系数 | 说明 |
|---|---|---|---|
| `irregular` | 12%（介于 follicular 和 luteal） | × 0.88 | 周期不稳定，按平均赤字处理；不再 phase 切换 |
| `peri_menopause` | 10% | × 0.90 | 围绝经期代谢变化，不激进 |
| `menopause` | 10% | × 0.90 | 绝经后；优先维肌防骨松，赤字慎重 |
| `breastfeeding` / `pregnant` | 0%（拒绝赤字） | × 1.0 或更高 | 仅推维持，且必须先咨询医生 |
| `medical` | 由医生决定 | — | 本 skill 不主动给数字 |

`status=regular` 用户继续看下表：

| Phase | Deficit | 摄入目标 = TDEE × 系数 | 说明 |
|---|---|---|---|
| `follicular` | 15% | × 0.85 | 减脂效率最高，按计划赤字执行 |
| `ovulatory` | 15% | × 0.85 | 同上 |
| `luteal` | 5-10%（默认 7%） | × 0.93 | 黄体期热量需求实际更高，赤字过大易暴食反弹 |
| `menstrual` D1-D3 | 0%（维持） | × 1.0 | 经期前 3 天按 TDEE 吃，不刻意赤字 |
| `menstrual` D4-D5 | 5% | × 0.95 | 出血明显减少后微赤字 |

**绝对底线**：摄入目标永远 ≥ **1200 kcal**。如算出来 < 1200（极小体型用户），按 1200 kcal 报，并标注"已触发安全底线，不再下调"。

**ED 信号触发**：参考 `cycle-rules.md §跨阶段共通原则`，立即停止赤字计算，仅报维持值。

例（接上面）：1976 × 0.85（follicular）≈ **1680 kcal/天**摄入目标 ；1976 × 0.93（luteal）≈ **1838 kcal/天**

## 3.5 三大营养素分配（每日饮食目标拆解）

摄入目标 kcal 算出来后，**必须**进一步拆成蛋白 / 脂肪 / 碳水 三个数字交给用户。

### 蛋白质（kcal/g = 4）

| 场景 | g/kg 体重 |
|---|---|
| 默认 | 1.2-1.4 |
| `luteal` 期 / `follicular`-`ovulatory` 力量训练为主 | 1.4-1.6 |
| `menopause` / `peri_menopause` | 1.4-1.8（防 sarcopenia） |
| 重训练日 | 1.6-2.0（力量增肌派推荐范围） |

例：58 kg 育龄女性，luteal 期 → 58 × 1.5 = **87 g 蛋白质 / 天 ≈ 348 kcal（占总摄入约 19%）**

### 脂肪（kcal/g = 9）

| 场景 | g/kg 体重 | 占总摄入 |
|---|---|---|
| 默认 | 0.8-1.0 | 25-30% |
| `luteal` / `peri_menopause`（激素友好） | 1.0-1.2 | 30-35% |
| 极低脂减脂尝试 | **不推荐**（< 20% 影响月经 + 激素） | — |

例：58 kg → 58 × 1.0 = **58 g 脂肪 / 天 ≈ 522 kcal（占约 28%）**

### 碳水（kcal/g = 4）

剩余 kcal / 4 = 碳水克数。**不戒碳水**——这是不节食派的核心理念，复合碳水（红薯 / 燕麦 / 糙米）优先。

例：1840（总目标） − 348（蛋白） − 522（脂肪） = 970 kcal → **243 g 碳水 / 天（占约 53%）**

### 输出格式（强制）

每次给摄入目标时，必须三大营养素**全部**列出：

```
摄入目标：约 1840 kcal
蛋白质：约 87 g  / 脂肪：约 58 g  / 碳水：约 243 g
```

### 月经期特例

`menstrual` D1-D3：碳水比平时增加 10-15%（缓解疲劳），相应蛋白脂肪略减比例。蛋白质底线仍 1.0 g/kg 不破。

## 3.6 饮水量计算与分时段建议

### 每日基础饮水量

```
基础 = weight_kg × 30-35 mL
训练日 +500-800 mL（按训练强度 / 时长 / 出汗）
气温高（夏季 > 28°C）+ 300-500 mL
luteal 期再 +300 mL（抗水肿，不要因为水肿就少喝水）
```

例：58 kg luteal 期训练日 → 58 × 33 + 500（训练）+ 300（luteal）≈ **2700 mL/天**

### 分时段建议（按 8 小时清醒分配）

| 时段 | 量 | 备注 |
|---|---|---|
| 起床后 5-10 min | 300-500 mL | 温水，唤醒代谢 |
| 上午（10-12 点） | 400-500 mL | 餐前 30 min 喝 200 mL 控餐量 |
| 训练前 30 min | 200 mL | 不要太多，避免训练时跑厕所 |
| 训练中 | 100-200 mL / 30 min | 小口频次，不要一次猛灌 |
| 训练后 30 min 内 | 300-500 mL | 补水分 + 助代谢 |
| 下午（14-17 点） | 400-500 mL | |
| 晚餐前 30 min | 200 mL | 控餐量 |
| 晚饭后到睡前 | ≤ 300 mL | **过多会夜醒**，影响睡眠 |

### 饮料替代建议

- 黑咖啡 / 茶 / 苏打水：可计入 50-70%（咖啡因有轻度利尿）
- 含糖饮料 / 奶茶 / 果汁：**不计入**（额外 kcal）
- 凉茶 / 熬煮的汤：可计入 100%
- 酒精：**不计入**且当天饮水量 +500 mL

### 警示

- 短时大量饮水（> 1L / 1 小时）有水中毒风险，**禁止**
- 月经期不要因为水肿就刻意少喝（反而促进水肿）
- 超过 35 mL/kg 没有额外收益，多了只是多上厕所

## 4. 训练燃烧 kcal 参考表（60 kg 女性 / 30 分钟基准）

| 类型 | kcal / 30 min | 备注 |
|---|---|---|
| 散步（5 km/h） | 90-120 | 月经期 / 主动恢复优选 |
| 快走（6-7 km/h） | 150-180 | 稳态有氧 |
| 慢跑（8 km/h） | 240-280 | 中强度有氧 |
| 跳绳（中速） | 250-300 | 高强度有氧，关节敏感慎用 |
| 爬楼（中速） | 200-240 | 通勤友好 |
| 游泳（自由泳中速） | 200-250 | 关节友好 |
| 骑行（中速） | 180-220 | 关节友好 |
| 力量训练（家用器械） | 120-180 | 包含组间休息 |
| 力量训练（健身房自由重量） | 150-220 | 包含组间休息 |
| HIIT | 240-350 | 短时高消耗 |
| 普拉提（标准强度） | 120-180 | |
| 阴瑜伽 / 流瑜伽 | 90-150 | |

**调整系数**（按用户体重线性缩放）：
- weight_kg / 60 = 调整系数
- 例：50 kg → 系数 0.83；70 kg → 系数 1.17

例：上面 58 kg 用户做 45 min 力量（家用） → (58/60) × (150 × 45/30) ≈ **218 kcal**

输出时一律说"约 220 kcal"，不要带小数。

### 4.5 训练分段消耗参考（强制按段给数字，不要只给总数）

LLM 在 Report 3 / Report 2 输出训练板块时，**必须把训练拆成 4 段**（热身 / 主体 / 拉伸 / 注意），并给每段独立的 kcal：

| 分段 | 典型时长 | 60 kg 基准 kcal | 备注 |
|---|---|---|---|
| 🔥 热身 | 5-10 min | 25-50 | 动态拉伸 / 开合跳轻版 / 弓步走 |
| 💪 主体 力量 | 30-45 min | 120-180 | 含组间休息；哑铃 +20% |
| 💪 主体 HIIT | 20-30 min | 200-300 | 高强度短时 |
| 💪 主体 稳态有氧 | 30-60 min | 150-300 | 取决于心率与类型 |
| 🧘 拉伸 | 5-8 min | 10-25 | 静态为主；最低消耗段 |

**输出格式（强制）**：
```
🔥 热身 [X] min：[1-2 句动作概要] · 约 [Z1] kcal
💪 主体 [Y] min：[动作清单] · 约 [Z2] kcal
🧘 拉伸 [W] min：[1-2 句动作概要] · 约 [Z3] kcal
合计训练消耗：约 [Z1+Z2+Z3] kcal
```

按 `weight_kg / 60` 缩放每段。

## 5. 每日输出格式（强制使用）

LLM 生成日计划或周计划时，每一天都要包含这一段。**不要省略**——这是用户每天能看到进度的关键反馈。

### 训练日格式

```
**周一 臀腿日（黄体期 D19，中等强度）**

📊 今日热量
- 摄入目标：约 1840 kcal（黄体期赤字 7%，TDEE 1976 × 0.93）
- 训练燃烧：约 220 kcal（45 min 力量，家用）
- 净赤字：约 130 kcal/天 ≈ 0.13 kg/周

🍱 饮食重点（参考 lifestyle-tips.md）
- 早餐 ~400 kcal：燕麦杯 + 茶叶蛋 ×2 + 黑咖啡
- ...

🏋️ 训练动作
- 自重深蹲 ×12 ×3 组 ![](URL)
- ...
```

### 休息日格式

```
**周日 全休 / 拉伸**

📊 今日热量
- 摄入目标：约 1840 kcal（黄体期）
- 训练燃烧：0（仅基础代谢）
- 净赤字：约 140 kcal/天

🧘 主动恢复（可选）
- 30 min 阴瑜伽或散步（不强制）
```

### 周表汇总格式（用户问"这周怎么吃"时）

```
**本周摄入目标：约 1840 kcal/天**（黄体期，赤字 7%）
**本周训练总消耗：约 700-900 kcal**
**预估周净赤字：约 1000-1200 kcal ≈ 0.13 kg 体重 / 0.05% 体脂**
```

不要承诺"一周瘦 X kg"。脂肪 1 kg ≈ 7700 kcal，用户体重周内波动大多是水分（黄体期尤甚），用以上"预估值"措辞即可。

## 6. 边界情况

| 场景 | 处理 |
|---|---|
| 用户没填 `body_fat_pct` | 不影响 BMR 计算，正常进行 |
| 用户没填 `weight_kg` 或 `height_cm`（必填字段缺） | 拒绝出 kcal 数字，提示"画像不全无法计算"，引导补全 |
| 用户体重远超 BMI 30 | 用当前体重算 BMR 即可；不要建议极端赤字 |
| 用户 `body_fat_pct` < 18 | 不再赤字，按维持值；触发 `cycle-rules.md` 健康警示 |
| 训练燃烧 > 摄入目标的 30% | 警告 "今日训练消耗偏高，建议加一份训练后蛋白" |
| menstrual 期任何赤字尝试 | 一律改成维持，不允许 deficit > 5% |
| 用户问 "今天吃了 X，还能吃多少" | 摄入目标 - 已吃，告知剩余预算；记录但不存入 profile（v0.0 不做食物日志） |

## 7. 用户教育（首次给 kcal 数字时夹一句）

第一次输出 kcal 数字时，加一句温和提示：

> "💡 这些数字是估算（BMR 公式 + 活动系数 + 阶段调整），用来给你方向，不是精确处方。体重周内波动 1-2 kg 多是水分，看 2-3 周的趋势更靠谱。"

同一会话内不重复。

## 8. 与其他文件的协作

- `cycle-rules.md` — phase 强度规则，**优先**于本文件的 deficit 计算
- `lifestyle-tips.md` — 单餐 kcal 区间（早 / 午 / 晚），用本文件的摄入目标统筹
- `weekly-split.md` — 训练时长，喂入本文件第 4 节算训练燃烧
- `exercises.md` — 单个动作示范，与 kcal 无直接关系
- `state/profile-schema.md` — 必填字段缺失时本文件无法运行
